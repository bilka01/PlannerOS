# PlannerOS

## Overview

PlannerOS is a local AI-powered productivity assistant.

The project combines ChatGPT with a lightweight Python application to automate daily planning.

ChatGPT is responsible for thinking and planning.

PlannerOS is responsible for executing those plans.

PlannerOS never generates plans on its own. It only interprets structured commands generated by ChatGPT and performs the requested actions.

The application is designed to be:

- Local-first
- Free to use
- Modular
- Easy to extend
- Independent of paid APIs

---

# Goals

PlannerOS should help users manage their day without requiring manual scheduling.

The user communicates naturally with ChatGPT.

Example:

> I have 5 free hours today.
> I want to:
>
> - study SQL
> - work on Power BI
> - apply for 3 jobs
> - go to the gym

ChatGPT produces:

1. A human-readable daily plan.
2. A structured PlannerOS command.

PlannerOS executes the command.

---

# Core Principles

## ChatGPT plans

PlannerOS executes.

The application never tries to "think."

Business logic belongs inside ChatGPT.

Execution logic belongs inside PlannerOS.

---

## Local First

PlannerOS should work completely offline whenever possible.

Cloud services should only be used when explicitly enabled by the user.

---

## Modular

Every feature lives in its own module.

Modules communicate only through the Dispatcher.

No module should directly depend on another module.

---

## Simple

Avoid unnecessary complexity.

Prefer small modules with a single responsibility.

---

# MVP (Version 0.1)

- Reading the clipboard
- Finding a Planner block
- Parsing JSON

No AI.

No scheduling logic.

# Planner Protocol

PlannerOS communicates with ChatGPT using a structured protocol.

Example:

```text
{
  "version": 1,
  "type": "daily_plan",
  "calendar": [],
  "tasks": [],
  "obsidian": {}
}
```

The protocol is versioned to ensure future compatibility.
PlannerOS parses this JSON into a validated PlannerCommand object.
After parsing, the dispatcher and handlers operate only on PlannerCommand.
